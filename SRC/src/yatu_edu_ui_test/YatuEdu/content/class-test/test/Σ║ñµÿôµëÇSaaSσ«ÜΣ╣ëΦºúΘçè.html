业务板块
每个业务板块由业务运营商签署SaaS平台入驻协议后授予一个业务代码，指定一个板块管理
人，如
1. MDR：船舶币的交易板块
2. SDCE：虚拟货币的交易板块
3. SGX：黄金币的交易板块
开通某个业务代码，每个交易板块由一个批发交易所和N个零售交易所构成
交易类型
1. 做市：得到交易所授权的做市商用户定时报买入、卖出价，不限数量，其他本交易所用
户只能在本交易所成交。做市商报价后可以确认是否成交散户提起 交易，如果不成交，
这个报价就要撤销，这个做市商必须另外报价。所有做市商的报价按照高低排序。
2. 撮合：全部非做市零售交易所合法用户都可以看到、报价、撮合成交
交易所
1. 批发交易所：每个业务板块只有一个，板块管理人设置交易所交易类型
2. 零售交易所：有无限多个，可向平台单独申请或者由板块管理人推荐，平台开通后授予
一个交易所代码，指定一个交易所管理人，板块管理人推荐的零售交易所默认可交易板
块批发交易所设定可共享的产品和交易对。交易所管理人可以申请加盟其他板块批发交
易所，经过其他板块批发交易所批准后可以可交易板块批发交易所设定可共享的产品和
交易对。零售交易所与批发交易所分享交易对的手续费。零售交易所管理人可以灵活选
定交易类型是做市还是撮合。
3. 批零兼营交易所：每个业务板块只有一个，板块管理人设置交易所交易类型，零售交易
所可以选择加盟，但是手续费没有分成，主要是零售交易所靠好的产品吸引流量，但是
批零兼营交易所自己也可以吸引流量，进行交易，这个交易所只能是撮合交易类型
定 义
产品与交易对
1. 由业务板块管理员录入基本信息，每个产品由平台统一编号（业务板块代号+顺序号）
2. 业务板块管理员录入交易对，并设置交易对是否可分享给加盟零售交易所
3. SaaS平台在产品池里面增加该业务板块管理员录入的所有产品和交易对，并对接相应
的公链和支付结算渠道
用户
1. 业务板块管理员：
1. 一级用户，负责管理该业务板块的交易所，并处理该交易所的用户开户申请
2. 定义交易所类型（做市、撮合）
3. 录入产品信息
4. 组成交易对
5. 设定交易板块指数
6. 为加盟商提交零售交易所开设申请，输入基本信息
2. 零售交易所管理员：
1. 二级用户，负责管理该零售交易所
2. 定义交易所类型（做市、撮合）
3. 给三级用户开户、KYC
4. 申请加盟、退出业务板块
5. 指定做市商（自己不一定做做市商，可以引入别的二级用户作为做市商），可以
指定多个做市商，竞争报价
用户
3. 散户：
1. 三级用户
2. 在某个零售交易所开户、KYC
3. 买卖
3. 做市商
3. 二级用户
4. 在某个批发交易所开户并且被某个零售交易所指定为做市商，可以提出
买卖指令，带数量的指令在批发交易所成交，不带数量的在零售交易所
披露，如果零售交易所有散户吃他的单子，他要么成交，要么重新报价
不成交。
5. 他能看到批发交易所的交易对价格，然后可以给零售交易所报价
显示信息
1. 行情信息：该交易所的交易对最新报价行情，做市模式的交易所，显示所有做市
商的报价
2. 指数信息：交易所开通业务板块的指数信息
1. 接受业务板块开户申请，生成业务板块代号
2. 指定该业务代号下的一级用户（业务板块管理员）
3. 业务管理员开设批发交易所，选择交易类型：做市、撮合
4. 业务管理员录入产品，交易对、设定指数组成
5. 为所有零售交易所的用户进行KYC
6. 为加盟商申请开设零售交易所
7. 业务板块管理员按照交易模式参与本批发交易所的交易
8. 按照业务板块的产品、交易对的交易信息得出清算报告

解析
• 0号店有无限货源（卖）
• 0号店总可以买（低于卖出价x%)
• 2~N号店做市商模式：
– 2级用户从一号店低价买入，在本店高于买入价挂单卖
– 2级用户从3级用户低价收购，在一号店高价卖出
– 3级客户只能看到店主的买卖价格，不能限价
– 做市商需要足够的资金，买入大批token
• 2~N号店撮合模式
– 客户随意报价
– “做市商”在本店报价的基础上报给1号店
– 客户如果不能自己之间撮合，与做市商撮合
– 客户只能看到本店的行情
– 做市商需要有算法，保证自己不亏
– 客户喜欢那种模式？
• 无论哪种模式，其实都有很多复杂的细节
